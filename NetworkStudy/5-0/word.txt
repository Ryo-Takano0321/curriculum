■ ワイヤレス
WLC→Wireless LAN controller 集中管理型のWLANにおいて複数のAPを一元管理するためのデバイス
集中管理型AP→アンテナの役割とデータ受信時の暗号化/複合処理だけを行いその他の設定はWLCが行う。中~大規模ネットワークに最適
自律型AP→AP単体で動作する。設定も各APごとに行う。小規模ネットワークに最適。
CSMA/CA→Carrier Sense Multiple Access with Collision Avoidance 無線LAN通信規格のIEEE802.11a/b/gの通信手順として採用されている。
        通信開始前に伝送媒体上に現在通信をしているホストがないかどうかを確認し通信を開始する。通信できる状態がわかってもランダム時間待機してからデータを送信する。
        無線LANではフレーム衝突を検知(collision detection)できないため、この手法で衝突を回避する。
SOHO→Small Office/Home Office　小さな事務所や自宅をオフィスとして働く職場形態
BSS→Basic Service Set　1つのAPとそのAPの電波内にいる配下の無線LANクライアントで構成されるネットワーク
　BSSID→無線LANにおけるネットワーク識別子の一つであり48ビットの数値。通常その無線ネットワークのAPのMACアドレスと同じものとなる。
ESS→Extended Service Set　複数のBSSで構成される無線LANのネットワーク
　ESSID→無線LANにおけるネットワーク識別子の一つであり最大32文字までの英数字を設定可能。無線LANを構成する機器（APや無線LAN端末）に設定する必要がある。
ローミング→複数のAPがある環境で、wi-fiに接続している端末が移動したときに自動的に近くのAPに接続が切り替わる仕組み。切り替わるAPのESSIDはすべて同じである必要がある。
CAPWAP→Control and Provisioning of Wireless Access Point WLCとAP間で通信制御を行うためのプロトコル。制御メッセージで使用するポート番号は5246、データで使用するポート番号は5247
　　　　機器間はトンネル接続を行う。制御メッセージは、AES-CCMPにより暗号化されるがデータはカプセル化のみ
■ FHRP　→　First Hop Redundancy Protocol デフォルトゲートウェイ冗長化のためのプロトコルの総称。複数台あるルータを仮想的に1つに見せることができる。
HSRP→Hot Standby Router Protocol デフォルトゲートウェイを冗長化するためのシスコ独自のプロトコル。
　　　　アクティブルータがARP応答する際に通知する「仮想MACアドレス」はデフォルトでは「0000.0c07.ac」までの値は常に同じであり、最後の1バイトはHSRPのグループ番号により値が変わる。
preempt→設定後や起動時の時差がある場合でもプライオリティ値が一番高いデバイスがアクティブルータになるようにできる機能
インターフェーストラッキング→HSPRを有効にしていないインターフェースがダウンしたときに、有効にしているインターフェースのプライオリティ値を小さくしてアクティブルータを切り替える機能
VRRP→Virtual Router Redundancy protocol デフォルトゲートウェイなどを冗長化するためのRFC3768で標準化されたプロトコル。マルチベンダー環境で実装可能。マスタールータとバックアップルータで構成される。
　　　マスタールータがARP応答する際に通知する「仮想MACアドレス」は0000.5e00.01xx までの値は常に同じであり最後の1バイト（8ビット）はVRRPのグループ番号で値が変わる。
GLBP→Gateway Load Balancing Protocol デフォルトゲートウェイなどを冗長化するためのシスコ独自のプロトコル。HSRPやVRRPとは異なり、1つのグループ内で利用可能なデフォルトゲートウェイを複数存在させることができ、
      負荷分散を実現できる。
アクティブルータ→仮想IP(仮想MAC)アドレスを宛先として転送されてきたトラフィックをルーティングするルータ
スタンバイルータ→アクティブルータがダウンした場合に仮想IP(MAC)アドレスの制御を引き継ぐルータ

■ QoS→Quality of Service ネット上で提供するサービス品質のこと。ネットワーク機器に実装することである特定の通信を優先して転送させたり、帯域幅を確保することができる。
モデル　IntServ(Integrated Services)→アプリケーションの通信フローごとに帯域を予約する方法。RSVPというプロトコルを使用する。大規模ネットワークには適していない。一般的に使用されていない。
 　   DiffServ(Differentiated Services)→トラフィックを分類、マーキング(優先度付け)、キューイング(トラフィックのキューへの振り分け)、スケジューリング(キューの優先度に応じたパケットの送出)してネットワーク機器ごとに管理。
　　　　　　　　　　　　　　　　　　　　 広く採用されている方式。一般的にQoSといえばこのモデルのことを指す。
     ベストエフォート→上記のアーキテクチャによるQoSが実装されていなかった場合、パケット内容やサイズに関係なく先着順にパケットを送出していくベストエフォート型(=FIFO)が提供される。
　　　　　　　　　　　 つまり、何も設定がなければQoSのデフォルトの動作として、全トラフィックはFIFOで処理される。
輻輳管理→パケットに割り当てられた優先順位に基づいて、トラフィックフロー（またはパケット）がインターフェイスに送信される順番を決定することにより、輻輳を制御できる機能
　FIFO →First In, First Out 先入れ先出しのキューイング方式。Ethernet LANインターフェースでデフォルトで有効になっている。パケットを先着順で処理する。
　PQ →Priority Queuing 各パケットに割り当てられたプライオリティに基づいてキューからパケットを転送するキューイング方式　high,medium,normal,lowの4つのキューが使用され、優先度の高いキューのパケットをすべて送出してからほかのパケットが送出される
　CQ →Custom Queuing 各キューを順番にサイクルして管理者が定義したバイト値のパケットを順番に転送していく方式。PQでは、インターフェースあたりに定義できるキューは最大4個だが、CQの場合は最大16個のキューを定義することができる。
　WFQ →Weighted Fair Queuing 送信元と宛先間で発生するトラフィックフローを動的に検知して、フローごとに個別のキューを管理する方式。自動的にキューが作成されるため、作成するための設定はいらない。優先度はIPパケットのToSフィールドにあるIP Precedence値を使用して決定する。
      IP Precedence値が大きいパケットほど優先して処理される。キューのウエイトはIP Precedence値が大きいほど小さくなり、キューのウエイトが小さい値ほど優先処理される仕組み。　　　
　CBWFQ →Class-Based WFQ　管理者が定義したクラスごとに個別のキューが割り振られるキューイング方式。各クラスごとに（キューごとに）最低保証帯域を指定できる。CQとWFQを組み合わせたような技術。
　LLQ →CBWFQの方式にPQの方式を追加したキューイング技術。CBWFQのように各キューの帯域幅を保証しながらも、PQのように特定トラフィックを最優先処理させることができる。現在の企業ネットワークで最も主流なQoSのキューイング技術
　IP Precedence→IPヘッダの中のToSフィールド(Type of Service:8bit)で定義されたIPパケットの優先度を表す値のこと。0~7の8段階あり値が大きいほどその優先度が高いことを意味する。Tos8bitのうちの3bitを使用して優先度を表す。
　DSCP→Differentiated Services Code Point　IP Precedenceと同様にIPパケットに優先度を付加するために使用。DSCPでは、diffservアーキテクチャで再定義されたToSフィールド（8bit）のうち6bitをその値に使用。
　　　　　DSCPでは「0～63の64段階」の優先度を定義することができることから、より細かく優先度付けが行われる。
　CoS→Class of Service　Ethernetフレームに設定する（マーキングする）ことができるEthernetフレームにおける通信の優先度を表す値のこと。IPパケット→IP PrecedenceまたはDSCP、Ethernetフレーム→CoS
　
輻輳回避→輻輳を発生させないようにする技術
　WRED→Weighted Random Early Detection 輻輳を回避するためにインターフェースのキューにたまってきたパケットをランダムに破棄する機能。テールドロップと違いパケットの優先度を見た処理を実現できる。
　テールドロップ→ルータでWREDを適用しない場合にデフォルトで適用されるパケットドロップのメカニズム。キューが満杯になると以降のキューに入らないパケットはすべて破棄される。
帯域制御→利用可能な帯域を制限する技術
　シェーピング制限レートを超えたパケットはI/Fのキューにバッファさせる。遅延が生じる恐れがある。遅延が許されない音声トラフィックには適していない。
　ポリシング→制限レートを超えたパケットはドロップさせる。またはパケットの優先度の変更を行う。

■ WAN
VPN→Virtual Private Network　仮想的なプライベートネットワーク接続。インターネットなどの公衆網を利用する場合でもIPsec等の高度なセキュリティを実装でき、安全に企業の拠点間通信を実現できる。大きくインターネットVPNとIP-VPNの2つに分類できる。 
インターネットVPN→インターネットなどの公衆網を利用したVPN。セキュリティプロトコルにIPsecを使用したIPsec-VPNと、SSLを使用したSSL‐VPNの2種類がある。
　サイト間VPN→ルーターでIPsecによる暗号化とカプセル化を行うので、クライアントPCは暗号化通信を意識することなく通信する。IPsecを実装したルータやセキュリティ製品はVPNゲートウェイと呼ばれる。
　クライアントVPN→PCにVPNクライアントソフトウェアをインストールして、そのPC上で暗号化とカプセル化を行いリモート接続先のVPNゲートウェイと通信を行う。
IP-VPN→MPLS-VPNのこと。通信事業者のプライベートIP網内で経路情報の探索にMPLSを採用したVPNのこと。MPLS→Multi-Protocol Label Switching ラベルと呼ばれるタグを使用したパケット転送技術。4byteのMPLSヘッダのみを参照したラベルスイッチングが行われるためIPルーティングより高速
IPsec→暗号化システムの技術により、ネットワーク層にてデータのセキュリティを保護するのに使用されるプロトコル。複数のプロトコルから構成される。
　AH→Authentication Header　パケットが改ざんされていないかどうか認証を行う。
　ESP→Encapsulated Sequrity Payload　パケットが改ざんされていないかどうか認証を行う。＋パケットのペイロード部の暗号化を行う。
　IKE→Internet Key Exchange　秘密鍵情報の交換を安全に行う。
GRE→Generic Routing Encapsulation　トンネルプロトコルの1つ。（トンネリング→あるトラフィックを別のプロトコルでカプセル化して伝送する技術。）任意のプロトコルのパケットをIPトンネル内でカプセル化する。
　　  暗号化機能を備えていないためルーティングセキュリティが保たれても、データそのものに対するセキュリティが保たれていないため、データの安全性を保ちたい場合はGRE over IPsecを使用する。ルーティングプロトコル使用可。
クラウド→コンピュータネットワークをベースとしたコンピューター資源の利用形態の1つ。自身のPCで管理していたソフトウェアやデータなどをインターネットなどを介してサービスの形で必要に応じて利用する方式。
　SaaS→Software as a Service　インターネット経由でソフトウェアパッケージを提供する利用形態。例）Google apps, Microsoft Online Services, Salesforce
　PaaS→Plattform as a Services　インターネット経由でアプリケーション実行用のプラットフォームを提供する利用形態。利用者がアプリを開発して稼働させるために必要な開発環境となるプラットフォームを提供する。
　　　　　例）AWSのAmazon S3やAmazon DynamoDB、Force.comプラットフォーム、Google App Engine、AppScale、Windows Azure
　IaaS→Infrastructure as a Service　インターネット経由でハードウェアリソース（CPU/メモリ/ストレージ）のITインフラを提供する利用形態。サーバ仮想化技術を利用してITインフラを提供、例）Amazon EC2
オンプレミス→クラウドを利用せず自社で保有、自社の設備で運用すること。要望に合わせて柔軟にカスタマイズ可能、社内のシステムとの連携や統合などの自由度が高いメリットがありますが初期費用が高額。

■セキュリティ
脆弱性→ソフトウェアの設計ミス、不具合などで発生する欠陥のこと
脅威→情報セキュリティを脅かしてシステムに対して危害を与える要因
マルウェア→不正で有害な動作を行う目的で作成された悪意のあるソフトウェアの総称
　ウイルス→自己伝染機能、潜伏機能、発病機能のいずれかを有する加害プログラムのこと。動的には活動せず、プログラムファイルからプログラムファイルへ静的に感染する。
　ワーム→自信を複製してほかのシステムに拡散する性質を持った独立したプログラムのこと。宿主となるファイルを必要とせずネットワーク経由でコンピュータに感染していく。
　トロイの木馬→表向きは無害だが、攻撃対象のデバイスに侵入後、攻撃を開始するプログラムのこと。
スプーフィング→ネットワーク上の別のデバイスまたはユーザになりすますこと。
　DoS攻撃→Denial of service　ネットワーク機器、サーバへのシステム攻撃を行い、正規ユーザへのサービス提供を不能にすること。具体的には、SYNフラッド攻撃、ICPMフラッド攻撃。smurf攻撃などがある。
   SYNフラッド攻撃→攻撃対象のサーバに対し、TCP接続要求のSYNパケットのみを大量に送り付ける攻撃。
   ICPMフラッド攻撃→攻撃対象のサーバに対し、ICPM Echo Requestを短時間で大量に送りつける攻撃。
   Smurf攻撃→攻撃対象のサーバに対し、踏み台のホストからICPM Echo Reply	を大量に送る攻撃のこと。
　DDoS攻撃→Distributed Denial of serviceDoS攻撃と似た手法を利用するが、より大規模で実行される。
　中間者攻撃→攻撃対象となる二者間の通信を、特別なソフトウェアを使用して不正に傍受、盗聴すること
ソーシャルエンジニアリング→システム管理者から話術や盗み聞きなどの社会的な手段によって、IDやパスワード情報を入手すること
ブルートフォースアタック→総当たり攻撃と呼ばれる手法。可能なパスワードを手あたり次第試す攻撃。
WPA→Wi-Fi Protected Access 無線の暗号化規格の一つ。WEPの脆弱性を改善した後継規格。暗号化アルゴリズムはTKIPによるRC4。PSK (Pre-Shared Key) 認証、またはIEEE802.1X認証を採用することができる
　WPAパーソナル→無線LANにおける暗号化や認証方法の一つ。暗号化規格としてWPAを使用し、PSK：事前共有鍵を使用して認証を行う方式
　WPAエンタープライズ→無線LANにおける暗号化や認証方法の一つ。暗号化規格としてWPAを使用し、認証サーバ（Radiusサーバ）を使用して認証を行う
サプリカント→IEEE802.1X認証を行うための３つの構成要素のうちの一つ。IEEE802.1Xにおけるクライアントまたはクライアントにインストールするソフトウェア。
オーセンティケータ→IEEE802.1X認証を行うための３つの構成要素のうちの一つ。サプリカントと認証サーバの仲介役となるネットワーク機器。
認証サーバ→IEEE802.1X認証を行うための３つの構成要素のうちの一つ。ユーザ認証を行うサーバのこと。IEEE802.1X/EAPに対応したRadiusサーバを使用する。
多要素認証→アクセス権限を得るために必要な本人確認のために2つ以上の要素で認証する方式。大きく下記3つがある。
　知識要素→ID/パスワード、PIN番号、秘密の質問
　所持要素→ワンタイムパスワード、ICカード、USBトークン、SMS認証、E-mail認証
　生体要素→指紋認証、顔認証、網膜認証、静脈認証
AAA→Authentication（認証）Authorization（認可）、Accounting（アカウンティング）の略称のことであり、これら3つの異なるセキュリティ機能を設定するためのアーキテクチャ上の枠組みのこと
　Authentication→認証　ユーザID、パスワード、デジタル証明書などの有効なクレデンシャル確認し正当なユーザであるかを決定する。
　Authorization→認可　認証に成功したユーザに対して提供する権限を制限する。
　Accounting→　認証に成功したユーザに対して情報を収集する。
RADIUS→Remote Authentication Dial In User Service　ネットワーク上のユーザ認証プロトコルの1つ。電話回線でダイヤルアップ接続の方式を利用するユーザに対してインターネット接続を実現するための認証プロトコルとして当初は使用されていた。
　　　　現在では無線LANや有線LANでのネットワーク接続時のユーザ認証のプロトコルとしても利用されている。認証システムはRadiusサーバ、Radiusクライアント、ユーザーの3つの要素で構成
