■ ネットワーク層
IP　→Internet Protocol OSI参照モデルではネットワーク層で,TCP/IPの階層モデルにおいてはインターネット層で動作するプロトコル
IPヘッダ　→データを送る宛先となるIPアドレスなどが含まれたヘッダ
ICMP　→Internet Control Message Protocol IPプロトコルにおいて、コンピュータ間での通信状態を確認するためのプロトコル
ICMPメッセージ　→大きく２種類　query: pingやtracerouteなどによる特定のノードに対する通信状態の確認、error:ノード間の通信で障害等が発生したときにその原因を送信元ノードに通知
IPアドレス　→インターネットやLANなどのIPネットワークに接続されたコンピュータなどに割り振られる識別番号。ネットワーク上の住所のようなもの
　オクテット　→情報量の単位　＝8ビット
　ネットワーク部　→IPアドレスのうち各ホストが所属するネットワークを示す部分
　ホスト部　→IPアドレスのうちそのネットワークに所属してる各ホストを示す部分
　クラス　→用途に分けて使用するアドレスの範囲を定義しているもの。A～E
　プライベートIPアドレス　→会社や家庭などのローカルで一意に割り振られるアドレス
　グローバルIPアドレス　→インターネットに接続している機器のに対して一意に割り当てられるアドレス
ネットワークアドレス　→ネットワーク自体を示すアドレス。ホスト部ビットがすべて0
ブロードキャストアドレス　→あるセグメントに接続されているすべてのホストにパケットを送信するためのアドレス。ホスト部ビットがすべて1
ループバックアドレス　→自分自身を示す仮想的なIPアドレス
サブネット　→クラスA～Cのネットワークをさらに複数に分割したネットワークのこと
サブネットマスク　→IPアドレス32ビットのうち何ビット分をネットワーク部とするかを表す、32ビットの数値
　プレフィックス表記　→IPアドレスにおいて何ビット目までがネットワーク部かを簡易的に表記したもの　ex.)/24
デフォルトゲートウェイ →宛先がわからないパケットの転送先となるIPアドレスのこと
IPv6　→IPアドレスの枯渇を解消するために開発された、128ビットのアドレス長をもつIPプロトコル。340澗個の割り当てが可能
　サブネットプレフィックス　→IPv4におけるネットワーク部に該当するもの
　インターフェイスID　→IPv4におけるホスト部に該当。通常MACアドレスより算出
 

ルータ　→コンピュータネットワークにおいてデータを2つ以上の異なるネットワーク間に中継する機器
　コンソール接続　→ルータのコンソールポートとPCをコンソールケーブルで接続すること
　VTY接続　→ルータなどの機器にネットワークを通じて別のコンピュータから操作や管理を行うために仮想的に接続すること
　AUX接続　→ルータのAUXポート(補助ポート)とモデムを接続し、電話回線経由で接続する方法
　ルーティングテーブル　→受信したパケットをどこに転送するべきかを決定するための宛先ルートの一覧情報
　　ネクストホップ　→次に転送する最も近いルータやインターフェース
　ARPテーブル　→ARPによって割り出したIPアドレスとMACアドレスの対応表
　　ARP　→Address Resolution Protocol IPアドレスからEthernetのMACアドレスを得られるプロトコル
　コアルータ　→通信キャリアやISPネットワークを中継するために用いられる。高速、大型
　センタールータ　→企業で中心的に利用されるルータ。拠点となる本社やデータセンター等企業で中心となる場所を接続
　エッジルータ　→企業で使用され、中心となる拠点を除く支店、営業所等の拠点間を接続するネットワークを構成
　ブロードバンドルータ　→家庭環境等でインターネット回線に接続することを目的として使用される
 

ルーティング　→宛先となるホストまでパケットを送信する際の最適経路を選択して送信すること
　直接接続　→ルータがそのネットワークに直接接続していること
　スタティックルーティング　→管理者が宛先への最適経路を手動で設定すること
　ダイナミックルーティング　→プロトコルによって動的にルートを決定すること
　　ディスタンスベクタ型　→ホップ数（経由するルータの数）により最適ルートを決定するアルゴリズム
　　リンクステート型　→各ルータがネットワーク全体の接続状況を把握したうえで最適ルートを決定するアルゴリズム
　　ハイブリッド型　→ディスタンスベクタとリンクステートの両方の機能を持ち合わせたもの、Cisco独自のdualというアルゴリズムで最適ルートを決定する
 

■ トランスポート層
コネクション型通信　→通信前に相手との間で仮想的な通信回路を確保する方式
　ACK　→二者間の通信において何らかの肯定的な応答をする際に送られる信号やパケット
コネクションレス型通信　→通信前に相手との間で事前のやり取りを行わない方式
　TCP　→Transmission Control protocol 信頼性の高い通信を行うためのプロトコル
　スリーウェイハンドシェイク　→コネクション型であるTCPにおける、データ転送前のコネクションの確立のこと
　Telnet　→ネットワークに接続された機器を遠隔操作するために使用するアプリケーション層プロトコル　TCPポート23
　HTTP　→HyperText Transfer Protocol Webサーバとクライアントのwebブラウザがデータを送受信するために使用するアプリケーション層のプロトコル　TCPポート80
　FTP　→File Transfer Protocol 特定のコンピュータ間でファイル転送するときに使用するアプリケーション層のプロトコル　TCPポート20、21
　SMTP　→Simple Mail Transfer Protocol 電子メールを送信するために使用するアプリケーション層のプロトコル　TCPポート25
TCPヘッダ　→TCPにおいて扱われるデータの先頭部分にある制御情報などを一定の形式で記述した領域
UDP　→　User Datagram Protocol TCPと比較して信頼性は高くないが、が速さやリアルタイム性が必要とされるときに使用されるプロトコル　コネクションレス型
　RTP　→Real-time Transport Protocol 音声や動画などのデータストリームをリアルタイムで再生するためのプロトコル　UDP
　SNMP　→Simple Networl Management Protocol TCP/IPネットワークに接続された機器に対し、ネットワーク経由で監視、制御するためのアプリケーション層プロトコル　UDPポート161
　DHCP　→Dynamic Host Configuration Protocol ネットワーク接続するのに必要なIPアドレスなどの情報を自動的に割り当てるアプリケーション層プロトコル　UDPポート68、67
　TFTP　→Trival File Transfer Protocol 特定のコンピュータ間でファイル転送するときに使用するアプリケーション層のプロトコル
          FTPとの違いは、ユーザー名、パスワード認証がなく、UDP上で動作している点　UDPポート69
　RIP　→Routing Information Protocol ディスタンスベクタ型のルーティングプロトコル　UDPポート520
UDPヘッダ　→UDPにおいて扱われるデータの先頭部分にある制御情報などを一定の形式で記述した領域
